# Colab 노트북 공유 설정 수정 가이드

## 문제 상황

"요청한 파일이 없습니다" 오류가 발생하는 이유:
1. 공유 설정이 "액세스 권한이 있는 사용자"로 되어 있음
2. 노트북이 삭제되었거나 이동됨
3. 노트북 ID가 잘못됨

## 해결 방법

### 1단계: Colab 노트북 확인

1. **Colab 노트북 직접 접속**
   ```
   https://colab.research.google.com/drive/1yJ-DOQaqh3ACIWdSGF4aBNtZ1OK5VAT5
   ```
   - 본인 계정으로 접속 가능한지 확인
   - 접속 안 되면 → 노트북이 없거나 ID가 잘못됨

### 2단계: 공유 설정 변경 (중요!)

1. **Colab 노트북 열기**
   - 위 URL로 접속

2. **공유 버튼 클릭**
   - 우측 상단 "공유" 버튼 클릭

3. **공유 설정 변경**
   - "일반 액세스" 섹션 찾기
   - 현재: "제한됨" 또는 "액세스 권한이 있는 사용자"
   - 변경: **"링크가 있는 모든 사용자"** 선택
   - 권한: **"뷰어"** 또는 **"편집자"** 선택
   - **"완료"** 클릭

### 3단계: 공유 설정 확인

1. **시크릿 모드로 테스트**
   - 브라우저 시크릿 모드 열기 (Ctrl + Shift + N)
   - 노트북 URL 접속:
     ```
     https://colab.research.google.com/drive/1yJ-DOQaqh3ACIWdSGF4aBNtZ1OK5VAT5
     ```
   - 노트북이 열리면 → ✅ 성공!
   - "요청한 파일이 없습니다" 오류가 나면 → 공유 설정 다시 확인

### 4단계: 노트북이 없다면

노트북이 삭제되었거나 없다면:

1. **새 노트북 생성**
   - https://colab.research.google.com 접속
   - "새 노트북" 클릭

2. **간단한 코드 작성**
   ```python
   import pandas as pd
   
   print("브릿지 모의고사")
   print("CSV 데이터를 로드하여 문제를 풀어보세요.")
   ```

3. **저장**
   - "파일" → "저장"
   - 제목: "브릿지 모의고사 - 1회차"

4. **공유 설정**
   - "공유" → "링크가 있는 모든 사용자"
   - 권한: "뷰어"

5. **새 노트북 ID 확인**
   - URL에서 `/drive/` 뒤의 ID 복사
   - `lib/config.ts`에 업데이트

## 여전히 안 되면

1. **노트북이 실제로 존재하는지 확인**
   - Google Drive에서 `.ipynb` 파일 찾기
   - 파일이 없으면 새로 생성

2. **다른 노트북으로 테스트**
   - 새 노트북 생성
   - 공유 설정 변경
   - 새 ID로 테스트

3. **노트북 ID 재확인**
   - URL에서 정확한 ID 복사
   - 공백이나 특수문자 없는지 확인
